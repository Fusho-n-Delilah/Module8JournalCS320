How can I ensure that my code, program, or software is functional and secure?
We ensure our code, Program, and software is functional and secure by testing it. This can include unit testing,
performance testing, integration testing, penetration testing, and a whole host of testing. It also starts with the
requirements and solution documents. We can review these documents before we ever write a line of code to ensure
developers are working with accurate information and developing exactly what is needed. 

How do I interpret user needs and incorporate them into a program?
It starts with a lot of meetings (or at least a few concentrated efforts) to determine the User's requirements.
Those requirements are then documented. A solution is proposed, and accepted. Following this, all those documents need reviewed.
These can be informal reviews, and even more formal reviews where team members act in the user persona, etc. to ensure the
needs will be met. These documents, once approved, become the basis of all user stories, and eventually this is the functionality
that is built into the program.

How do I approach designing software?
As stated above we first get the requirements from the business, propose a solution, and then get that solution approved.
We then get a UX/UI design that outlines what it should look like, and the user experience as they navigate the application.
At that point a backlog of user stories are created. The first two steps mentioned are how the software is designed, roughly.
